# README — Documentation

## But
Cette documentation est organisée pour séparer clairement:
- les **références contractuelles** (spécifications validées)
- les **documents de travail** (brouillons, audits)
- l’**historique** (archives)

## Où trouver quoi ?

- Références: `docs/reference/`
- Travail: `docs/work/`
- Archives: `docs/history/`
- Index: `docs/_meta/INDEX.md`

## Règle d’or
Tout nouveau document commence dans `docs/work/`.
